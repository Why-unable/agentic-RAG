# 后退式提问 (Step-Back Prompting) Skill

本技能让智能体学会在用户问题“过于微观”时往后退一步（Take a step back）。特别是当提问包含罕见日期、深层代码报错或刁钻的边缘案例，且直接检索失败时，需要靠抽象化抽取宏观规律来进行辅助打底。

## 何时使用本技能 (When to Use)

当你遇到以下情况时使用此技能：
- 查询中塞满了极度具体的微观参数、私人代码、特定日期或特定的边缘场景（例：“为什么我的 Python 代码在计算 2024 年 2 月 29 日这天报错了？”）。
- 你怀疑知识库内只留存了宏大的通用规范、API 原则或通用法则，并不会记录用户特有的私人业务数据。
- 采用直接满文搜索，因包含了过多的特定具体数字或名称干扰项，导致匹配为零。

## 强制处理步骤 (Required Steps)

1. **识别核心概念 (Identify the Core Concept)**：穿越层层报错与无关的业务细节，看透用户真正碰壁的技术点、物理法则或业务通用规定是什么。
2. **生成退化问题 (Generate Step-Back Question)**：基于核心概念编写一个高维度、抽象性的概括问题。它应当去探求那个领域的“一般规律”。
3. **执行双轨检索 (Execute Dual Retrieval)**：
   - 将原版非常“具体刁钻”的问题拿去搜索（作为兜底尝试）。
   - 将刚才“抽象概括”的后退问题拿去搜索（获取大口径的原理资料）。
4. **融汇回答 (Synthesize)**：抽取原理文档中的根本法则，反过来套用于用户所提供的那个超精细特例上，推演出符合逻辑的解答。

## 输出格式与执行日志 (Output Format & Execution Logging)

应用该技能时，需记录问题被如何完成剥离与抽象：

```markdown
**[应用技能: 后退式提问 Step-Back Prompting]**
- 原始用户查询: "<非常具体的场景提问>"
- 提炼出的核心概念: "<技术盲点/统管规则>"
- 抽象/退化查询: "<关于这个核心规律的高维提问>"
```

**示例**:
- 原始查询: "为什么我的 Python 代码在处理 2024年2月29日 的日期格式时报错了？"
- 提炼出的核心概念: "Python datetime 库关于闰年异常处理与界限规则。"
- 抽象/退化查询: "Python 中 datetime 库处理闰年的常见错误与规范是什么？"

用大口径的闰年文档打底，你就能轻松推演解释由于年份不合规导致 2 月 29 日超出边界的那个具体的代码 Bug。
