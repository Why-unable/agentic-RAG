# 假设性文档嵌入 (HyDE) Skill

本技能旨在解决向量检索空间中的结构不匹配问题。短小的“问题（Question）”与详实的“答案（Answer）”在语义向量上的重合度往往不高。通过预先生成一篇假设性回答去进行匹配，能大幅提高目标文段的搜索精准度。

## 何时使用本技能 (When to Use)

当你遇到以下情况时使用此技能：
- 用户的查询高度开放、宽泛或非常抽象（例：“对刚入职的应届生有哪些合规红线要求？”，“遇到内存溢出如何排查？”）。
- 你预判知识库中的答案是以列表、政策文件或详尽段落形式呈现，而不是直白的 Q&A 问答对。
- 直接将“问题本身”输入向量库检索产生的结果全不相关，或相似度过低。

## 强制处理步骤 (Required Steps)

1. **草拟假答案 (Draft a Hypothetical Answer)**：运用大模型现有的预训练常识与直觉，为该问题写一篇貌似完美（甚至可能是幻觉）的陈述性回答。确保其采用与预期知识库文档一致的语调（如官方陈述语气）。
2. **作为搜索基准 (Use as Search Vector)**：坚决**不使用**用户抛出的简短提问，转而将这篇**生成的假答案长文本**投入向量数据库进行搜索。
3. **事实矫正 (Grounding - 严重警告)**：当 RAG 系统将知识库中真正的相似文档取回之后，**必须完全抛弃**你刚才假造的答案。合成最终反馈时，必须百分之百仅针对真正取回的文档材料进行回复，绝不夹带编造的幻觉细节。

## 输出格式与执行日志 (Output Format & Execution Logging)

应用本技能时，你的推理环节必须留下以下日志：

```markdown
**[应用技能: 假设性文档 HyDE]**
- 原始提问: "<用户的原始查询>"
- 生成的假设性靶向文档:
  """
  <写 2 到 3 句话的假设性详实回答，模拟知识库内文章的语调>
  """
- 行为: 已将假设性靶向文档提交至向量检索...
```

**示例**:
- 原始提问: "对于刚入职的校招生，有什么必须注意的合规红线？"
- 生成的假设性靶向文档: "作为校招生，必须严格遵守公司的合规红线。主要包括数据隐私保护、反腐败与反商业贿赂条例、知识产权保密协议（NDA）以及职场社交媒体发言规范。严禁任何形式的内幕交易和利益冲突。"

这篇利用常识虚拟出的大段文本将成为一块强力“磁铁”，去知识库中大量吸附具有相同词法与句法的真实合规守则。
