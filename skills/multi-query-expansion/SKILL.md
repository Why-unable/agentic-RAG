# 语义扩展与多路查询 (Multi-Query / Expansion) Skill

本技能提供了克服“词汇鸿沟”的操作指南。当用户使用的词汇与知识库内部使用的专业术语不严格一致时，这一技能至关重要。

## 何时使用本技能 (When to Use)

当你遇到以下情况时使用此技能：
- 用户的提问使用了高度口语化、非正式或宽泛的表述。
- 你推测目标知识库采用了特定的行业术语、技术黑话说辞或协议名称（例：用户说“投屏”，知识库里可能是“多屏互动”或“DLNA协议”）。
- 首次检索返回的结果质量极低或找不到相关信息。

## 强制处理步骤 (Required Steps)

1. **意图分析 (Analyze Intent)**：深入理解用户查询的核心意图。
2. **变体生成 (Generate Paraphrases)**：必须生成不少于 3 至 5 个同等含义的查询变体。变体形式需要涵盖：
   - 同义词、近义词与常用俗语。
   - 更正式的描述与官方术语。
   - 特定行业协议或技术专有名词。
3. **并行检索 (Execute Parallel Searches)**：将原始问题连同生成的所有变体一起（或通过多路并发）提交到检索引擎。
4. **去重合并 (Deduplicate)**：对全部回收的文档结果进行排重和汇总，然后再进行信息提炼生成最终回答。

## 输出格式与执行日志 (Output Format & Execution Logging)

应用该技能时，需要在工作日志中输出你的扩展词生成结果：

```markdown
**[应用技能: 多路查询扩展 Multi-Query Expansion]**
- 用户原始意图: "<意图>"
- 生成的查询变体:
  1. "<变体 1 - 专业术语>"
  2. "<变体 2 - 同义替换>"
  3. "<变体 3 - 结果导向>"
```

**示例**:
- 用户原始意图: 怎么投屏？
- 生成的查询变体:
  1. "无线显示设置"
  2. "屏幕镜像教程"
  3. "DLNA连接方式"

通过覆盖不同视角的表述，能够最有效地防止由于单纯的“用词差异”而导致的任务失败。
